# 合同管理系统（本地落地版）

这是一个可直接本地运行的企业合同管理系统演示版，围绕“起草 -> 审批 -> 生效 -> 履约 -> 收付款 -> 归档 -> 审计”主线实现了可操作界面和 REST API。

## 当前已实现

- 登录鉴权（内置管理员、法务、财务账号）
- 用户管理、角色权限、模块授权、重置密码、首次登录强制修改密码
- 合同驾驶舱：合同总额、审批中、生效合同、到期提醒、待回款、风险合同
- 合同台账：查询、筛选、新建、编辑、详情侧边栏
- 合同编号自动生成：`HT-年份-序号`
- 新建 / 编辑合同时上传附件，并保存到本地数据文件
- Excel / CSV 台账导入：按表头自动映射合同字段并预览回填
- Word / PDF / TXT 合同解析：已集成到“新建合同”弹窗中，自动抽取合同名称、金额、日期、付款节点、履约节点并回填
- OCR 识别入口：当前保留在“新建合同”弹窗中；机器可直接处理文本型 PDF，图片 OCR 预留本机 OCR 引擎扩展位
- 付款节点配置：支持在合同里维护付款计划
- 履约里程碑配置：支持在合同里维护执行节点
- 审批中心：默认主线为“部门审批 -> 法务审核 -> 财务审核 -> 领导终审 -> 电子签章”
- 合作方档案、模板中心
- 收付款计划、提醒预警、归档借阅
- 审计日志留痕
- 本地 JSON 数据持久化，便于继续扩展为 MySQL / Redis / MinIO 架构

## 技术方案

- 前端：原生 HTML + CSS + JavaScript 单页应用
- 后端：Node.js 原生 HTTP 服务
- 数据：`data/db.json`
- 部署：本地 Node 直接启动

这样做的目的是先把业务系统跑起来，方便快速评审交互和流程，再平滑升级到 Vue3 + SpringBoot + MySQL 正式架构。

## 启动方式

```bash
cd contract-management-system
npm start
```

启动后访问：

```text
http://127.0.0.1:3060
```

## 默认账号

- 超级管理员：`admin / Admin@123456`
- 法务：`legal / Legal@123456`
- 财务：`finance / Finance@123456`

新增账号默认密码规则：`账号 + @123456`

首次登录或管理员重置密码后，系统会强制要求用户修改一次密码。
